# Loop Retrospective

## Cuándo usar

Después de loops fallidos, costosos o repetidos.

## Procedimiento

1. Revisar métricas del loop.
2. Identificar patrón de fallo.
3. Determinar si faltó contexto, spec, test, eval o permiso.
4. Actualizar documentación si aplica.
5. Actualizar evals si hubo fallo IA.
6. Registrar aprendizaje para Engram.
7. Actualizar grafo en Graphify si cambió arquitectura o dependencia.
