# Loop Review

## Cuándo usar

Antes de aceptar el resultado de un loop.

## Procedimiento

1. Revisar objetivo original.
2. Revisar spec o task.
3. Revisar archivos modificados.
4. Revisar tests/evals ejecutadas.
5. Revisar intentos y reparaciones.
6. Revisar scope creep.
7. Revisar riesgos.
8. Emitir resultado: approve, changes required, rollback.
