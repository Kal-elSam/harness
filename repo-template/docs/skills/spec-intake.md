# Spec Intake Skill

## When to use

Use at the start of any non-trivial task.

## Procedure

1. Read the user request.
2. Infer likely affected system areas.
3. Identify ambiguity.
4. Identify risks.
5. Run spec complexity classification.
6. Create the appropriate spec file under `docs/specs/`.

## Output

```txt
Spec path:
Spec level:
Reason:
Missing information:
Safe assumptions:
Blocked or ready:
```
