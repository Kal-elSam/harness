# TDD Skill

Use this skill for behavior-changing code.

## Source of truth

- `AGENTS.md`
- `docs/ai/test-driven-development.md`
- `docs/ai/testing.md`

## Workflow

1. Identify expected behavior.
2. Write failing test first.
3. Confirm it fails for the right reason.
4. Implement minimal change.
5. Run related tests.
6. Run full validation if needed.

## Output

```txt
Failing test:
Implementation:
Validation:
Coverage impact:
```
